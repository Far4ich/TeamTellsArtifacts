
**Статус:** принято

### Контекст:
Выбор фронтенд-фреймворка приложения TeamTells определяет подход к созданию интерфейса приложения

### Рассмотренные варианты:
1. Vue.js - фреймворк, реализованный как дополнительная разметка для HTML
2. React - библиотекой пользовательского интерфейса
3. Angular - фреймворк, предоставляющий широкий и полный спектр инструментов для разработки веб-приложений

### Решение:
- Принято решение использовать Angular

### Обоснование:
- Удобство выделения **общих/типовых компонентов** и их переиспользования
- Angular структурирует код через **модули**, что удобно с точки зрения реализации проекта **(на каждую сущность в проекте - свой модуль)**
- **Наличие опыта** у ряда разработчиков

### Недостатки:
- **Высокий порог вхождения** (сложная структура фреймворка)
- **Сложность отлова ошибок**, связанных со стилями/компонентами
- **Трудозатратное проектирование веб-приложения**

### Последствия:
- Следует воспитать культуру обмена опытом между техлидом/опытными разработчикам и менее знакомыми с Фреймворком участниками фронтенд команды (например отдельная тема в телеграм-канале)
- Следует согласовать convention по коду с заказчиками и составить чек-лист по Code Review
- Раннее выделение типовых компонентов и стилей должно упростить работу менее опытных разработчиков
